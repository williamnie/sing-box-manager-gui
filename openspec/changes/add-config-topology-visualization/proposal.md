# Proposal: 增加 sing-box 工作拓扑可视化

## Why

当前“配置预览”主要以 JSON 文本和基础统计呈现，能够提供完整信息，但在以下场景可读性较弱：

- 用户需要快速理解“流量从哪里进、如何匹配、最终从哪里出”
- 用户需要排查路由链路（规则命中、默认出口、策略组）
- 用户需要识别配置异常（引用不存在、孤立出站、回退路径不清晰）

为降低理解门槛并提升问题定位效率，需要提供一套结构化的拓扑流程图视图。

## What Changes

本提案新增“工作拓扑”能力，基于现有配置预览数据构建图形化展示：

1. 新增拓扑视图入口（建议在“设置 -> 配置”相关区域）
2. 将配置映射为三层主拓扑：上游接入（Inbounds）→ 内部路由（Route/Rule/Policy）→ 出口（Outbounds）
3. 展示默认回退路径（final outbound）与关键路由关系
4. 提供基础交互（节点详情、链路高亮）
5. 对异常拓扑关系给出可见告警（例如引用不存在的 outbound）

## Scope

### In Scope

- 使用现有 `config preview` 数据构图，不新增后端持久化结构
- 实现可视化布局与链路信息展示
- 支持大多数常见 sing-box 配置路径（inbounds/route/outbounds/final）

### Out of Scope

- 不做实时流量统计叠加（如每条链路吞吐）
- 不改动 sing-box 配置生成逻辑
- 不引入复杂图编辑能力（仅展示与定位）

## Impact

- 前端：新增拓扑数据转换层与图形渲染组件
- 交互：新增配置理解与诊断入口，降低配置排障成本
- 风险：需要处理不同配置结构差异，避免展示错误链路

## Risks & Mitigations

- 配置结构差异导致拓扑不完整
  - 通过“部分可视化 + 告警提示”策略保证可用性
- 大配置渲染性能下降
  - 使用分层布局、节点折叠和渲染节流控制复杂度
- 用户误解“逻辑路径”和“运行时命中路径”
  - 在 UI 中明确“此图为静态配置拓扑，不代表实时命中统计”

## Success Criteria

- 用户可在一个界面内看清入口、规则层和出口层关系
- 对缺失引用、悬空出站等配置问题有明确视觉提示
- 在 1000+ 节点配置下仍能保持可操作性（可滚动/可折叠/不卡死）
